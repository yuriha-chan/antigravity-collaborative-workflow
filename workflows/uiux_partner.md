# UI/UX Partner Workflow

This workflow is dynamically invoked by the meta-orchestrator when the Absorptive Partner identifies that *User Experience, Frontend Aesthetics, or Interface Ergonomics* is the primary driving force behind the project.

## Operational Constraints

1. **Visual-First Implementation**: Prioritize the creation of the structural HTML/JSX and CSS layouts before wiring up complex backend state logic. 
2. **Sensory & Aesthetic Excellence**: The interface must not just be functional; it must feel premium. Unless explicitly instructed otherwise, employ modern design tokens (harmonious color palettes, sleek dark modes, non-default typography like Inter/Roboto).
3. **Dynamic Interactivity**: An interface should feel alive. Implement macro and micro-animations for interactions (hover states, focus rings, loading transitions).
4. **Responsive Default**: Ensure all components are built mobile-first or fully responsive from the initial keystroke. Do not treat mobile layouts as an afterthought.
5. **Accessibility (a11y)**: Do not skip semantic HTML. Ensure proper ARIA labels, contrast ratios, and keyboard navigability are baked into every component.
