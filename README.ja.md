# Antigravity Collaborative Workflow: Absorptive Partner

**対話を最優先とするAIコーディングワークフロー**

このリポジトリには、アンチグラビティ向けに設計された、**人間中心の対話的ワークフロー群**が含まれています。
これにより、AIは従来の「タスク実行者」から「協働型設計者」へと根本的に変革されます。

## 理念：考えてからキーボードを打て

現代のソフトウェア開発において、迅速かつ効率的に*間違ったもの*を構築することは、最も洗練された失敗の方法です。
世の中のAIコーディングエージェントは生き急ぎすぎていています。プロンプトを受け取ると即座にコード生成に走り、アーキテクチャの不整合、肥大化したコードベース、要件の逸脱を招きます。このような失敗を避けるために、AIと人間のコミュニケーションを健全化するための、賢いワークフローが必要です。

**吸収型パートナー**哲学は3つの柱で構成されます：
1. **対話を最優先：** 「吸収フェーズ」では、暗黙の前提や、設計の思想、何がプロジェクトの範囲外の事項になるかついて、傾聴・探求・明確化のための質問を行います。破壊的なツール呼び出しやコード生成は禁じられています。AIは、大学のゼミのような知的なトーンで発言するように指定されています。
2. **暗黙知の可視化：** 人間のプロンプトを文字通りの仕様として扱うのではなく、ワークフローはそれを出発点として位置付けます。構造的な対話フローを活用し、ユーザーが明示的に述べていない隠れた複雑性を明らかにします。
3. **厳格な整合性確保：** AIは、ユーザーとの議論をまとめた合意事項を文章に書き起こし、ユーザーの承認を得てから実装に取り掛かります。この文章Alignment Summaryでは、ユーザーの発言を直接引用し、「Universal Language」（共有用語集）を確立することで、「統計的にまれな」革新的なアイデアを、確実な参照点として焼きこみます。

## 機能概要

システムは、中核となる「メタオーケストレーター」と、`.agents/workflows/` に配置された複数の高度に専門化された実行サブプロンプトで構成されています。

*   **`absorptive_partner.md`**: 中央のオーケストレーター。無限吸収ループ、整合性チェックポイント（フェーズ2）、合成（フェーズ3）を管理します。アラインメント完了後、プロジェクトの優先度を評価し、専門サブプロンプトへ実行をルーティングする。

**実行サブプロンプト（スペシャリスト）：**
*   **`tdd_partner.md`**: 厳格なレッド・グリーン・リファクタリングサイクルを強制。ターミナル出力で検証可能なテスト失敗を確認するまで、実装コードの記述を明示的に拒否する。
*   **`secops_partner.md`**: ゼロトラスト原則を注入。実行前の脅威モデリングを義務付け、最小権限に焦点を当て、OWASPトップ10脆弱性（LLM固有のプロンプトインジェクションを含む）を積極的に軽減。
*   **`mvp_partner.md`**: 市場投入速度を最適化。機能に厳格な数値的制限（最大3～5つのコアタスク）を設け、管理サービスを活用することで、徹底的な優先順位付けを強制します。
*   **`uiux_partner.md`**: 感覚的・美的卓越性を最優先し、コードの最初の行からビジュアルファースト実装、レスポンシブレイアウト、ユニバーサルアクセシビリティ（a11y）を強制します。
*   **`gradual_restructuring.md`**: 深いレガシーコードと極端な技術的負債に対処するため、不可能なグローバルな書き換えを試みるのではなく、AIに防御的な「腐敗防止レイヤー」の構築を教えます。
*   **`visualize_logic.md`**: `Mermaid.js` を通じて曖昧なロジックの視覚的マッピングを強制する特化ツール。

**調査の方法論:**
*   **`web_survey.md`**: 対象を絞ったウェブ調査を実施するための厳密かつ形式化された「計画、承認、実行」ワークフロー。調査戦略をアーティファクトとして出力し、ユーザーの承認を経てから検索を実行します。プロキシ検索語も利用した高度かつ広範なサーベイを自動化し、Webの情報汚染による低品質な結果を防ぎます。さらに、検索品質のレポートが提示され、さらなる調査戦略へのイテレーションも射程にはいります。

## 使用方法
1. `workflow/` および `rules/` ディレクトリをワークスペースの `.agents/` ディレクトリに配置します。
2. 既存のアイデア（メモ、チャット会話の抜粋、概念コードや文書など）がある場合は、ワークスペースディレクトリに配置します。
3. `absorptive_partner.md`ワークフローを開始します。例：`/absorptive_partner [目的]のアプリケーションを作成したい。ログファイル@log.txtの会話記録をみて。`と指示します。（Antigravityがワークフロー文書を認識しない場合、`@[.agents/workflows/absorptive_partner.md]`で起動可能）
日本語を用いて会話したい場合は、会話やTask.mdなどを日本語で記述するように依頼してください。

## 注意事項
Claudeモデルはワークフロー文書内の「急がないでください」という指示を非常に忠実に守ります。長時間の議論に疲れた場合は、タスクリストを参照して自ら話題を転換してください。フェーズ1で「タスク複製」指示は、繰り返しstandpointsを生成して、無限のデザイン議論を可能にすることを意図していますが、タスクリストの内容や順序が混乱する可能性があります。その場合は手動で整理してください。

Geminiモデルを使用する場合、フェーズ1では会話のペースが速くなります。議論を継続させるには、`@phase1_standpoints_generation.md`を使用してエージェントに追加の視点を生成するよう指示できます。
